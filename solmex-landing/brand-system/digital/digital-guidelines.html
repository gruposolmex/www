Border Radius System

Rounded Corners

Consistent border radius values create a cohesive, modern interface feel while maintaining the industrial precision of the brand.

/* Border Radius Tokens */
--radius-sm: 4px; /* Small elements, chips */
--radius-md: 8px; /* Buttons, inputs */
--radius-lg: 12px; /* Cards, containers */
--radius-xl: 16px; /* Modals, large cards */
--radius-full: 9999px; /* Pills, avatars */

Elevation & Shadows

Shadow System

Shadows create visual hierarchy and indicate elevation. Use sparingly to avoid visual clutter.

/* Shadow Tokens */
--shadow-sm: 0 1px 2px rgba(10, 10, 10, 0.05);
--shadow-md: 0 4px 6px rgba(10, 10, 10, 0.1);
--shadow-lg: 0 10px 15px rgba(10, 10, 10, 0.15);
--shadow-xl: 0 20px 25px rgba(10, 10, 10, 0.2);
--shadow-glow: 0 0 20px rgba(255, 148, 59, 0.3);

Interaction Design

Hover States

Interactive elements should provide clear visual feedback. Use subtle transforms and shadows to indicate interactivity.

Animation Timing

Consistent easing functions create smooth, natural-feeling interactions.

/* Timing Function */
--ease-default: cubic-bezier(0.4, 0.0, 0.2, 1);

/* Durations */
--duration-fast: 150ms;
--duration-normal: 200ms;
--duration-slow: 400ms;

Best Practices

Digital Design Guidelines
DO
  • Use the 8px baseline grid consistently
  • Maintain sufficient touch targets (44x44px min)
  • Ensure WCAG AA contrast ratios
  • Design mobile-first, enhance for desktop
  • Use subtle animations for feedback
  • Keep interactions under 400ms
DON'T
  • Mix spacing values outside the system
  • Use shadows excessively
  • Create jarring or sudden transitions
  • Ignore keyboard navigation
  • Use auto-playing audio or video
  • Sacrifice performance for aesthetics
